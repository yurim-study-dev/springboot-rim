## 학습 주제
타임리프(Thymeleaf) 뷰 템플릿 + Model 데이터 바인딩 + 게시판 CRUD 화면 구현

> 실습 파일: `spring-board/src/main/java/net/likelion/bebc25/board02/`  
> 템플릿 파일: `spring-board/src/main/resources/templates/board/`

## 어제(board01) vs 오늘(board02) 차이

어제는 자바 코드 안에 HTML을 문자열로 때려박았음:
```java
// board01 — 끔찍한 방식
@ResponseBody
public String getBoardList() {
    return "<html><body><table>..." + "...";  // 자바 안에 HTML
}
```

오늘은 HTML 파일을 따로 분리하고 데이터만 넘겨줌:
```java
// board02 — 깔끔한 방식
public String getBoardList(Model model) {
    model.addAttribute("posts", getPosts());
    return "board/list";  // templates/board/list.html 파일로 연결
}
```

---

## 뷰 렌더링 흐름

```
컨트롤러 → "board/list" 반환
    ↓
ViewResolver → templates/board/list.html 파일 찾기
    ↓
타임리프 엔진 → Model 데이터 + HTML 결합
    ↓
완성된 HTML → 브라우저 전송
```

컨트롤러는 뷰 이름만 알려주면 됨. 어떤 파일인지 직접 찾지 않음.

## Model 객체로 데이터 전달

```java
@GetMapping("/list.html")
public String getBoardList(Model model) {
    List<PostDto> posts = getPosts();
    model.addAttribute("posts", posts);  // key: "posts", value: 리스트
    return "board/list";
}
```

- `model.addAttribute("키", 값)` → HTML 파일에서 `${키}`로 꺼내 씀
- `@Slf4j` → `log.debug(...)` 로 콘솔에 디버그 로그 출력 (Lombok)

## 타임리프 기본 문법

### 네임스페이스 선언 (필수)

```html
<html lang="ko" xmlns:th="http://www.thymeleaf.org">
```

### th:text — 값 출력

```html
<td th:text="${post.title}">샘플 제목</td>
<!-- 렌더링 결과: <td>1번 게시글</td> -->
```

태그 안에 써둔 텍스트(`샘플 제목`)는 서버 없이 HTML 파일을 직접 열었을 때 보이는 샘플 데이터. 서버 실행 시엔 실제 데이터로 대체됨.

### th:each — 반복문

```html
<!-- list.html 실습 코드 -->
<tr th:each="post : ${posts}">
    <td th:text="${post.id}">3</td>
    <td>
        <a th:href="@{detail.html(id=${post.id})}" th:text="${post.title}">게시글 제목</a>
    </td>
    <td th:text="${post.author}">작성자</td>
    <td th:text="${#temporals.format(post.createdAt, 'yyyy-MM-dd HH:mm:ss')}">2026-07-13</td>
</tr>
```

### th:href — 링크 (URL에 파라미터 붙이기)

```html
<!-- 쿼리 파라미터: detail.html?id=1 -->
<a th:href="@{detail.html(id=${post.id})}">제목</a>

<!-- 경로 변수: posts/1/edit -->
<a th:href="@{posts/{id}/edit(id=${post.id})}">수정</a>
```

### 유틸리티 객체

```html
<!-- 날짜 포맷 (#temporals) -->
<td th:text="${#temporals.format(post.createdAt, 'yyyy-MM-dd HH:mm:ss')}">2026-07-13</td>
```

### 삼항 연산자 (조건에 따라 다른 텍스트)

```html
<!-- write.html — 등록이면 "게시글 등록", 수정이면 "게시글 수정" -->
<h1 th:text="${postForm.id == 0} ? '게시글 등록' : '게시글 수정'">게시글 등록</h1>
<button th:text="${postForm.id == 0} ? '등록' : '수정'">등록</button>
```

## 폼 데이터 바인딩 (th:object, th:field)

등록/수정 폼을 하나의 HTML 파일(`write.html`)로 재사용:

```html
<!-- write.html 실습 코드 -->
<form th:action="@{{action}(action=${postForm.id == 0} ? 'write' : 'edit')}"
      method="POST"
      th:object="${postForm}">

    <input type="hidden" th:field="*{id}">  <!-- id 숨김 필드 -->

    <input type="text" th:field="*{title}" placeholder="제목을 입력하세요">
    <input type="text" th:field="*{author}" placeholder="작성자">
    <textarea th:field="*{content}"></textarea>

</form>
```

- `th:object="${postForm}"` → 이 폼이 `postForm` DTO와 연결됨
- `th:field="*{title}"` → `id="title" name="title" value="..."` 자동 생성
- `*{필드명}` → `th:object`로 지정한 객체 안의 필드를 단축 참조

컨트롤러에서 빈 DTO를 넘겨줘야 폼이 에러 없이 뜸:
```java
// 등록 화면 (새 게시글 → id=0인 빈 객체)
@GetMapping("/write.html")
public String getWriteForm(Model model) {
    model.addAttribute("postForm", new PostDto());
    return "board/write";
}

// 수정 화면 (기존 데이터 채워서)
@GetMapping("/edit.html")
public String getEditForm(@RequestParam("id") int id, Model model) {
    PostDto post = getPosts().get(id - 1);
    model.addAttribute("postForm", post);
    return "board/write";  // 같은 파일 재사용!
}
```

## HTTP 파라미터 받는 방법

```java
// 쿼리 파라미터: /detail.html?id=1
@GetMapping("/detail.html")
public String getDetail(@RequestParam("id") int id, Model model) {
    PostDto post = getPosts().get(id - 1);
    model.addAttribute("post", post);
    return "board/detail";
}

// 폼 전체를 DTO로 한 번에 받기
@PostMapping("/edit")
public String editPost(@ModelAttribute PostDto post) {
    updatePost(post);
    return "redirect:detail.html?id=" + post.getId();
}

// 파라미터를 하나씩 받기
@PostMapping("/write")
public String writePost(@RequestParam("title") String title,
                        @RequestParam("content") String content,
                        @RequestParam("author") String author) {
    PostDto post = new PostDto(title, content, author);
    savePost(post);
    return "redirect:list.html";
}
```

## 게시판 CRUD 흐름 정리

| 기능 | HTTP | URL | 처리 |
|------|------|-----|------|
| 목록 조회 | GET | `/02/board/list.html` | 전체 목록 → list.html |
| 상세 조회 | GET | `/02/board/detail.html?id=1` | 1번 글 → detail.html |
| 등록 화면 | GET | `/02/board/write.html` | 빈 폼 → write.html |
| 등록 처리 | POST | `/02/board/write` | 저장 후 → redirect: list.html |
| 수정 화면 | GET | `/02/board/edit.html?id=1` | 기존 데이터 → write.html 재사용 |
| 수정 처리 | POST | `/02/board/edit` | 수정 후 → redirect: detail.html |
| 삭제 처리 | POST | `/02/board/delete` | 삭제 후 → redirect: list.html |

## 배운 점

1. ✅ 어제(board01): 자바 코드 안에 HTML 직접 작성 → 오늘(board02): HTML 파일 분리 + 타임리프
2. ✅ 컨트롤러는 뷰 이름(`"board/list"`)만 반환 → ViewResolver가 파일 경로 찾아줌
3. ✅ `model.addAttribute("키", 값)` → HTML에서 `${키}`로 꺼냄
4. ✅ `th:each` — 리스트 반복, `th:text` — 값 출력, `th:href` — 링크 생성
5. ✅ `#temporals.format(...)` — 날짜 포맷 유틸리티
6. ✅ `th:object` + `th:field` — 폼과 DTO 자동 연결, `id/name/value` 자동 생성
7. ✅ 등록 화면과 수정 화면을 `write.html` 하나로 재사용 (`id==0`이면 등록, 아니면 수정)
8. ✅ `@RequestParam` — 쿼리 파라미터 개별 수신, `@ModelAttribute` — 폼 전체를 DTO로 한 번에 수신
9. ✅ POST 처리 후 `redirect:` 붙이면 이중 등록 방지

## 오늘 핵심 정리

```
타임리프 기본 문법
  xmlns:th="http://www.thymeleaf.org"  ← 필수 선언
  th:text="${변수}"     ← 값 출력
  th:each="item : ${list}" ← 반복
  th:href="@{url(key=${값})}" ← URL + 파라미터
  th:if / th:unless    ← 조건부 렌더링
  #temporals.format()  ← 날짜 포맷

폼 바인딩
  th:object="${postForm}"  ← 폼과 DTO 연결
  th:field="*{title}"     ← id/name/value 자동 생성

파라미터 수신
  @RequestParam("id")  ← 쿼리 파라미터 하나씩
  @ModelAttribute      ← 폼 전체 → DTO 한 번에

등록/수정 화면 재사용
  id == 0 → 등록 화면
  id != 0 → 수정 화면 (기존 데이터 채워서)
```
